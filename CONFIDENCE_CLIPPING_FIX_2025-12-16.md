# 🎯 Confidence Clipping Fix - Исправление завышенной уверенности

**Дата:** 16 декабря 2025, 11:30 UTC  
**Проблема:** Avg Confidence = 92.5% (слишком высокий для реального рынка)  
**Причина:** Агрессивное масштабирование Raw Confidence → Trading Confidence

---

## 🔍 Анализ проблемы

### Симптомы
- **Avg Confidence: 92.5%** - нереально высокий для крипторынка
- **Trading Conf: 99-100%** - при Raw Conf 65%
- Подозрение: один из агентов выдаёт 100% уверенности

### Диагностика

**1. Проверка scale_confidence() в futures_brain.py:**

```python
# БЫЛО (агрессивное масштабирование):
def scale_confidence(self, raw_confidence: float) -> float:
    if raw_confidence >= 0.65:
        return 100.0  # ❌ Слишком агрессивно!
    else:
        scaled = ((raw_confidence - 0.30) / 0.35) * 70 + 30
        return min(100.0, max(30.0, scaled))
```

**Примеры масштабирования (БЫЛО):**
- Raw 30% → Trading 30%
- Raw 50% → Trading 70%
- Raw 60% → Trading 94%
- Raw 65% → Trading **100%** ❌

**2. Проверка статистики:**

```python
# Строка 327 futures_brain.py
self.stats['avg_confidence'] = (self.stats['avg_confidence'] * (n-1) + trading_conf) / n
```

Статистика считается от **trading_conf** (масштабированного), а не raw_conf!

**3. Реальные данные из логов:**

```
Raw Conf: 65% → Trading Conf: 100%
Raw Conf: 52% → Trading Conf: 73%
Raw Conf: 26% → Trading Conf: 30%
Raw Conf: 0%  → Trading Conf: 30%
```

**Вывод:** При Raw Conf 65% (максимум от LSTM) → Trading Conf 100% → Avg Confidence завышается!

---

## ✅ Решение: Confidence Clipping

### Изменения в futures_brain.py

**scale_confidence() - сбалансированное масштабирование:**

```python
def scale_confidence(self, raw_confidence: float) -> float:
    """
    Smart Scaling: масштабирование ML confidence (BALANCED)
    
    Raw ML выдаёт 0.30-0.65 (LSTM часто даёт 0.30 для HOLD)
    Масштабируем сбалансированно (не слишком агрессивно):
    
    Формула: trading_conf = (raw - 0.30) / 0.35 * 55 + 30
    - raw 0.30 -> 30%
    - raw 0.50 -> 61%
    - raw 0.60 -> 78%
    - raw 0.65 -> 85% (MAX - не даём 100%!)
    """
    if raw_confidence <= 0.30:
        return 30.0
    elif raw_confidence >= 0.65:
        return 85.0  # MAX 85% (было 100%)
    else:
        # Сбалансированное масштабирование 0.30-0.65 -> 30-85
        scaled = ((raw_confidence - 0.30) / 0.35) * 55 + 30
        return min(85.0, max(30.0, scaled))
```

**Новые примеры масштабирования:**
- Raw 30% → Trading 30%
- Raw 50% → Trading 61%
- Raw 60% → Trading 78%
- Raw 65% → Trading **85%** ✅ (было 100%)

## 🎯 Почему это важно

### Проблемы завышенной уверенности

1. **Искажение статистики** - Avg Confidence 92% не отражает реальную ситуацию
2. **Ложное чувство безопасности** - кажется что сигналы очень надёжные
3. **Неправильный Dynamic Leverage** - может использовать слишком высокое плечо
4. **Сложность анализа** - непонятно какие сигналы действительно сильные

### Преимущества Confidence Clipping

1. **Реалистичная оценка** - 85% max = "очень хороший сигнал", но не "идеальный"
2. **Правильная статистика** - Avg Confidence отражает реальное качество сигналов
3. **Лучший Risk Management** - не переоцениваем надёжность сигналов
4. **Честность перед собой** - крипторынок непредсказуем, 100% уверенности не бывает

## 📊 Мониторинг

### Что проверять

```sql
-- Avg Confidence (должен быть 55-70%)
SELECT 
    ROUND(AVG(
        CASE 
            WHEN extra_data->>'trading_confidence' IS NOT NULL 
            THEN (extra_data->>'trading_confidence')::numeric 
            ELSE 0 
        END
    ), 1) as avg_trading_conf
FROM trades 
WHERE status = 'CLOSED' 
  AND market_type = 'futures' 
  AND entry_time > NOW() - INTERVAL '24 hours';

-- Распределение confidence
SELECT 
    CASE 
        WHEN (extra_data->>'trading_confidence')::numeric < 40 THEN '30-40%'
        WHEN (extra_data->>'trading_confidence')::numeric < 50 THEN '40-50%'
        WHEN (extra_data->>'trading_confidence')::numeric < 60 THEN '50-60%'
        WHEN (extra_data->>'trading_confidence')::numeric < 70 THEN '60-70%'
        WHEN (extra_data->>'trading_confidence')::numeric < 80 THEN '70-80%'
        ELSE '80%+'
    END as conf_range,
    COUNT(*) as count
FROM trades 
WHERE status = 'CLOSED' 
  AND market_type = 'futures' 
  AND entry_time > NOW() - INTERVAL '24 hours'
GROUP BY conf_range
ORDER BY conf_range;
```
