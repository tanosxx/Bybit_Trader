# Demo API Limitations

## Проблема

При тестировании на Demo API (`api-demo.bybit.com`) обнаружены ограничения:

### ❌ Не работают приватные endpoint:
1. `/v5/execution/list` - Trade History (retCode 10004 - signature error)
2. `/v5/position/closed-pnl` - PnL History (retCode 10004 - signature error)
3. `/v5/asset/transfer/*` - Wallet Transactions (404 - endpoint not found)

### ✅ Работают публичные endpoint:
1. `/v5/market/kline` - Historical Klines ✅
2. `/v5/market/tickers` - Current Prices ✅

## Причина

**Demo API имеет ограниченный функционал!**

Согласно документации Bybit:
- Demo API предназначен для тестирования торговли
- Не все приватные endpoint доступны в Demo
- Исторические данные (trade history, PnL) могут быть недоступны

## Решение для ML проекта

### Вариант 1: Использовать только публичные данные (РЕКОМЕНДУЕТСЯ)
**Для обучения LSTM модели нам нужны:**
- ✅ Исторические свечи (OHLCV) - **РАБОТАЕТ!**
- ✅ Технические индикаторы (RSI, MACD, BB) - рассчитываем сами
- ✅ Временные фичи (hour, day_of_week) - рассчитываем сами

**Этого достаточно для ML модели!**

### Вариант 2: Переключиться на реальный API
- Использовать `api.bybit.com` вместо `api-demo.bybit.com`
- Получить доступ к полным историческим данным
- **НО:** Нужен реальный аккаунт с историей торговли

### Вариант 3: Использовать альтернативные источники
- Скачать исторические данные с CryptoDataDownload
- Использовать Binance API (больше исторических данных)
- Импортировать данные из CSV
