# 🔄 SYNC SERVICE v2.0 - Полная защита от фантомных позиций

## 📊 Проблема

**Фантомные позиции** - позиции которые закрыты на бирже (по SL/TP), но остаются открытыми в БД.

### Почему возникают?

1. **Задержка синхронизации:** Sync Service работал каждые 30 секунд
2. **Неточное сопоставление:** Сравнивал только по символу, не учитывал quantity
3. **Нет расчёта PnL:** При закрытии фантомов не рассчитывался PnL и комиссии
4. **Нет обучения ML:** Результаты фантомных позиций не использовались для обучения

### Последствия

- ❌ Расхождение БД и биржи (было 6 в БД, 3 на бирже)
- ❌ Неправильная статистика (Win Rate, PnL)
- ❌ Dashboard показывает несуществующие позиции
- ❌ ML модель не обучается на всех результатах

## ✅ РЕШЕНИЕ: Sync Service v2.0

### Ключевые улучшения

**1. Точное сопоставление позиций**
```python
# Уникальный ключ: (symbol, side, quantity)
key = (trade.symbol, trade.side.value, trade.quantity)

# Проверяем точное совпадение
if key not in exchange_positions:
    # ФАНТОМНАЯ ПОЗИЦИЯ!
```

**Было:** Сравнивали только по символу → могли быть дубликаты  
**Стало:** Сравниваем по (symbol, side, quantity) → точное совпадение

**2. Расчёт PnL и комиссий**
```python
# Получаем текущую цену
current_price = await self.api.get_ticker(trade.symbol)

# Рассчитываем PnL
if trade.side.value == 'BUY':
    pnl = (current_price - trade.entry_price) * trade.quantity
else:  # SELL
    pnl = (trade.entry_price - current_price) * trade.quantity

# Рассчитываем комиссии
entry_value = trade.entry_price * trade.quantity
trade.fee_entry = entry_value * settings.estimated_fee_rate

exit_value = current_price * trade.quantity
trade.fee_exit = exit_value * settings.estimated_fee_rate

# Закрываем с полными данными
trade.pnl = pnl
trade.exit_price = current_price
trade.status = 'CLOSED'
```

**Было:** Закрывали с PnL=0, без комиссий  
**Стало:** Полный расчёт PnL и комиссий как при обычном закрытии

**3. Обучение ML модели**
```python
# Обучаем Self-Learning модель на результате
if trade.ml_features:
    learner = get_self_learner()
    result = 1 if pnl > 0 else 0  # Win/Loss
    learner.learn(trade.ml_features, result)
```

**Было:** Фантомные позиции не использовались для обучения  
**Стало:** Каждая фантомная позиция обучает ML модель

**4. Ускоренная синхронизация**
```python
await asyncio.sleep(15)  # Каждые 15 секунд (было 30)
```

**Было:** 30 секунд → позиции могли закрыться и стать фантомами  
**Стало:** 15 секунд → быстрее обнаруживаем закрытые позиции

## 🔧 Технические детали

### Алгоритм работы

```
1. Получить позиции с биржи (Bybit API)
   ↓
2. Создать словарь: {(symbol, side, qty): position_data}
   ↓
3. Получить открытые позиции из БД
   ↓
4. Для каждой позиции в БД:
   ├─ Создать ключ (symbol, side, qty)
   ├─ Проверить наличие в словаре биржи
   ├─ Если НЕТ → ФАНТОМНАЯ ПОЗИЦИЯ:
   │  ├─ Получить текущую цену
   │  ├─ Рассчитать PnL
   │  ├─ Рассчитать комиссии
   │  ├─ Закрыть позицию в БД
   │  └─ Обучить ML модель
   └─ Если ЕСТЬ → Обновить quantity (если изменился)
   ↓
5. Сохранить изменения в БД
   ↓
6. Вывести статистику
```

### Структура данных

**Exchange Positions (с биржи):**
```python
{
    ('BTCUSDT', 'BUY', 0.05): {
        'symbol': 'BTCUSDT',
        'side': 'BUY',
        'size': 0.05,
        'entry_price': 42000.0,
        'leverage': '5',
        'unrealised_pnl': 15.50
    }
}
```

**DB Trades (из БД):**
```python
[
    Trade(
        id=16078,
        symbol='ETHUSDT',
        side='SELL',
        quantity=0.06,
        entry_price=3247.61,
        status='OPEN'
    )
]
```

### Логика сопоставления

```python
# Для каждой позиции в БД
for trade in db_trades:
    # Создаём ключ
    key = (trade.symbol, trade.side.value, trade.quantity)
    
    # Проверяем наличие на бирже
    if key not in exchange_positions:
        # ФАНТОМ - закрываем с расчётом PnL
        close_phantom_position(trade)
    else:
        # Существует - обновляем если нужно
        update_position_if_changed(trade, exchange_positions[key])
```
