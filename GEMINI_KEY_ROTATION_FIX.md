# Gemini API Key Rotation Fix - 6 декабря 2025

## 🐛 Проблема

**Обнаружено:** Ключи Gemini API ротировались **на каждый запрос**, а не **при исчерпании квоты**.

### Старая логика (НЕПРАВИЛЬНО):
```python
# Переключаемся на следующий ключ для следующей попытки
self.current_gemini_key_index = (self.current_gemini_key_index + 1) % len(self.gemini_keys)

# Делаем запрос
response = requests.post(url, ...)
```

**Что происходило:**
1. Запрос #1 → Key #1 → Success → **переключаемся на Key #2**
2. Запрос #2 → Key #2 → Success → **переключаемся на Key #1**
3. Запрос #3 → Key #1 → Success → **переключаемся на Key #2**
4. ...

**Результат:**
- ❌ Оба ключа исчерпывались **одновременно**
- ❌ Вместо 3,000 requests/day получали только 1,500
- ❌ Квота заканчивалась в 2 раза быстрее

---

## ✅ Решение

### Новая логика (ПРАВИЛЬНО):
```python
# Делаем запрос
response = requests.post(url, ...)

if response.status_code == 200:
    # Успех - НЕ переключаем ключ, продолжаем использовать текущий
    return text

elif response.status_code == 429:
    # Quota exceeded - ПЕРЕКЛЮЧАЕМСЯ на следующий ключ
    self.current_gemini_key_index = (self.current_gemini_key_index + 1) % len(self.gemini_keys)
    continue
```

**Что происходит теперь:**
1. Запрос #1 → Key #1 → Success → **остаёмся на Key #1**
2. Запрос #2 → Key #1 → Success → **остаёмся на Key #1**
3. ...
4. Запрос #1500 → Key #1 → **Quota exceeded** → **переключаемся на Key #2**
5. Запрос #1501 → Key #2 → Success → **остаёмся на Key #2**
6. ...
7. Запрос #3000 → Key #2 → **Quota exceeded** → **переключаемся на Key #1** (но он тоже исчерпан)

**Результат:**
- ✅ Используем **ВЕСЬ** лимит первого ключа (1,500 requests)
- ✅ Потом используем **ВЕСЬ** лимит второго ключа (1,500 requests)
- ✅ Итого: **3,000 requests/day** вместо 1,500
- ✅ Квота заканчивается в 2 раза медленнее

## 🔧 Изменения в коде

### Файл: `core/strategic_brain.py`

**Изменено:**
1. Переключение ключа **ТОЛЬКО** при ошибках (429, 400, 403, timeout)
2. При успехе (200) - **НЕ** переключаем ключ
3. Добавлены комментарии о логике ротации

**Ключевые изменения:**
```python
if response.status_code == 200:
    # ...
    if text:
        print(f"   ✅ Gemini API success (key #{key_index + 1})")
        # НЕ переключаем ключ при успехе - продолжаем использовать текущий
        return text

elif response.status_code == 429:
    print(f"   ⚠️  Gemini key #{key_index + 1}: Quota exceeded, switching to next key...")
    # ВАЖНО: Переключаемся на следующий ключ при исчерпании квоты
    self.current_gemini_key_index = (self.current_gemini_key_index + 1) % len(self.gemini_keys)
    continue
```
