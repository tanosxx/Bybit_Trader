# RSS Feed Parsing Fix - 5 декабря 2025

## Проблема

В логах бота появлялись предупреждения RSS парсинга:
```
⚠️ RSS Warning for https://www.coindesk.com/arc/outboundfeeds/rss/: <unknown>:2:0: syntax error
⚠️ RSS Warning for https://bitcoinmagazine.com/.rss/full/: <unknown>:2:751: not well-formed (invalid token)
```

## Причина

Некоторые RSS фиды имеют некорректный XML, что вызывает `bozo` флаг в feedparser.
Эти ошибки **некритичны** - данные всё равно парсятся, но логи засоряются предупреждениями.

## Решение

Улучшена обработка ошибок в `core/news_brain.py`:

**Было:**
```python
if feed.bozo:
    # feedparser обнаружил проблему с фидом
    print(f"⚠️ RSS Warning for {url}: {feed.bozo_exception}")
```

**Стало:**
```python
# Проверяем наличие записей
if hasattr(feed, 'entries') and feed.entries:
    return feed.entries

# Если нет записей, но есть bozo_exception - это критическая ошибка
if feed.bozo and not feed.entries:
    # Логируем только если действительно нет данных
    exception_str = str(feed.bozo_exception)
    # Игнорируем некритичные XML ошибки (syntax error, not well-formed)
    if 'syntax error' not in exception_str and 'not well-formed' not in exception_str:
        print(f"⚠️ RSS Warning for {url}: {exception_str}")

return []
```

## Логика

1. **Если есть данные** → возвращаем их (игнорируем bozo)
2. **Если нет данных И есть bozo_exception:**
   - Проверяем тип ошибки
   - Игнорируем "syntax error" и "not well-formed" (некритичные)
   - Логируем только критичные ошибки
